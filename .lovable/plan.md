
# Fix Weld Rod, Wall Coving Calculations, and Per-Room Waste Control

## Problems Identified

### 1. Weld Rod Showing 555.9m Instead of ~27m
The seam lines generated by the strip plan algorithm are deliberately extended 10x beyond the room bounding box for visual rendering/clipping purposes. However, the weld rod calculation uses these raw extended coordinates to compute length, resulting in massively inflated values.

**In `greedyStrip.ts` line 292:**
```text
const extensionFactor = 10;
const extensionX = bbox.width * extensionFactor;
```

This means a seam that should be ~8m long is stored as ~80m+ in the seam line coordinates.

### 2. Wall Coving Not Adjusting Drop Lengths
Currently, wall coving is treated as a separate linear accessory (perimeter x price). In reality, when coving is selected, each vinyl strip must be cut longer to run up the walls on both ends. For example:
- Room width: 5m, coving height: 100mm
- Each drop should be: 5m + 0.1m + 0.1m = 5.2m
- With 10% waste: 5.2m x 1.10 = 5.72m

This needs to modify the strip plan calculation, not just add a separate line item.

### 3. Weld Rod Should Include Perimeter When Coving Is Enabled
Weld rod is used to seal seams. When wall coving is installed, the perimeter junction between the floor vinyl and the coving also needs welding. So:
- **Without coving**: Weld rod = total seam length only
- **With coving**: Weld rod = total seam length + perimeter (net of doors/excluded walls)

### 4. No Per-Room Waste Percentage Control
Waste is currently defined on the material specs and can only be overridden at the report level by material. Users need to set waste per room since room geometry complexity varies.

## Solution

### Step 1: Fix Seam Length Calculation

**File: `src/lib/rollGoods/greedyStrip.ts`**

Add a new field `seamLengthPerSeamMm` to `StripPlanResult` that stores the actual (unextended) seam length. Each seam's true length equals the room dimension in the strip direction (`roomLength` variable, already calculated). Store `totalSeamLengthMm` on the result:

```text
totalSeamLengthMm = numberOfSeams x roomLength
```

**File: `src/lib/rollGoods/types.ts`**

Add to `StripPlanResult`:
- `totalSeamLengthMm: number` -- actual clipped seam length total
- `totalSeamLengthM: number` -- same in meters

### Step 2: Fix Weld Rod Calculation

**File: `src/lib/accessories/calculations.ts`**

Update `calculateSeamLengthFromPlan()` to use the new `totalSeamLengthM` field instead of computing from extended seam line coordinates.

Update `calculateWeldRod()` to:
1. Get actual seam length from `stripPlan.totalSeamLengthM`
2. When coving is enabled on the same room, add the net perimeter (perimeter minus doors minus excluded walls)
3. Accept `room`, `scale` parameters to compute perimeter when needed

**File: `src/lib/reports/calculations.ts`**

Update `calculateAccessoryCosts()` weld rod section (lines 209-223) to use `stripPlan.totalSeamLengthM` instead of computing from raw seam coordinates.

### Step 3: Adjust Strip Lengths for Wall Coving

**File: `src/lib/rollGoods/types.ts`**

Add to `StripPlanOptions`:
- `covingHeightMm?: number` -- when set, each strip length is extended by 2x this value

**File: `src/lib/rollGoods/greedyStrip.ts`**

In `calculateStripPlan()`, after computing `baseStripLength`:
- If `options.covingHeightMm > 0`, add `2 * covingHeightMm` to each strip's length
- This represents the vinyl running up both walls the strip spans between
- The waste factor is then applied on top of this extended length (already handled)

**File: `src/pages/ProjectEditor.tsx`**

When calculating strip plans (line 611), pass `covingHeightMm` from the room's accessories:
```text
covingHeightMm: room.accessories?.coving?.enabled ? (room.accessories.coving.heightMm || 100) : 0
```

**File: `src/lib/reports/calculations.ts`**

Same adjustment in `calculateRoomCost()` (line 294) -- pass coving height to strip plan options.

### Step 4: Per-Room Waste Percentage

**File: `src/lib/canvas/types.ts`**

Add to `Room` interface:
- `wastePercent?: number` -- per-room waste override (undefined = use material default)

**File: `src/components/editor/TakeoffPanel.tsx`**

Add an inline waste percentage editor in the room row (expanded section):
- Small input field showing current waste % (room override or material default)
- "Apply to all" button that sets the same waste % on all rooms sharing the same material
- Display format: a compact number input with "%" suffix

**File: `src/pages/ProjectEditor.tsx`**

Pass room-level waste override to strip plan calculation:
```text
wasteOverride: room.wastePercent  // undefined falls back to material default
```

**File: `src/lib/reports/calculations.ts`**

In `calculateRoomCost()`, check room-level waste override first, then material default:
```text
const wastePercent = room.wastePercent ?? getWastePercent(material.specs);
```

### Step 5: Update AccessoriesPanel Display

**File: `src/components/editor/AccessoriesPanel.tsx`**

- Weld Rod section: show breakdown "Seams: X.Xm + Perimeter: X.Xm = Total: X.Xm" when coving is enabled
- Wall Coving section: show note that drop lengths have been extended (e.g., "Drops extended by +200mm for 100mm coving")
